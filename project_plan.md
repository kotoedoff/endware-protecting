# Endware Cyber Security Telegram Bot: Project Plan

This document defines the functional requirements and logic for the open-source, self-hosted Telegram security bot.

The project is split into four main modules:
1. **Channel Protection (Часть 1: Канал)**
2. **Chat/Group Protection (Часть 2: Чаты)**
3. **AI Security Analysis & Automation (Часть 3: ИИ)**
4. **Configuration & Customization (Часть 4: Конфигурация)**

---

## 🛡 Part 1: Channel Protection (Канал часть)

Based on the core threats to Telegram channels, the scope of this module is focused on two primary defensive systems:

### 1. Анти-накрутка подписчиков (Anti-Bot Flood)
* **Детекция всплесков (Rate Limiting)**:
  * Бот отслеживает частоту запросов на вступление (Join Requests). Если скорость вступления превышает установленный лимит (например, более `X` человек в минуту, где `X` настраивается), бот переходит в режим "Тревога".
* **Анализ профилей (Heuristics Check)**:
  * При обработке заявок бот анализирует профиль каждого юзера:
    * *Имя/Никнейм*: Проверка по регулярным выражениям на наличие признаков ботоводства (например, повторяющиеся подряд символы вроде `TTTTT`, сгенерированные случайным образом комбинации).
    * *Заполненность профиля*: Оценивается отсутствие аватарки, пустое описание (Bio) и отсутствие юзернейма.
* **Автоматическое реагирование**:
  * В обычном режиме: Бот одобряет только тех, кто прошел базовую проверку.
  * В режиме "Тревоги": Бот автоматически отклоняет и блокирует всех участников, подпадающих под признаки ботов, снижая нагрузку на модераторов.

### 2. Анти-спам и Защита от взлома админов (Rogue Admin Protection)
* **Мониторинг публикаций (Content Watchdog)**:
  * Каждый новый пост или его изменение в канале проверяется ботом на стоп-слова, наличие ссылок (фишинг, реклама) и медиафайлов.
* **Экстренное реагирование (Panic Protocol)**:
  * *Снятие прав*: Если админ публикует подозрительный контент (NSFW, ссылки на наркотики, массовый спам), бот проверяет свои права. Если у бота есть право *«Назначать администраторов»*, он мгновенно отзывает все права у данного администратора.
  * *Очистка контента*: Бот удаляет все посты, опубликованные скомпрометированным админом за последний час.
  * *Уведомление владельца*: Бот отправляет владельцу канала (Creator) в ЛС экстренное сообщение о блокировке администратора.

## 💬 Part 2: Chat/Group Protection (Чаты)

Защита супергрупп и чатов от спамеров, ботов, вредоносного контента и социальной инженерии.

### 1. Анти-спам для обычных пользователей (User Anti-Spam)
* Фильтрация классического флуда (частые сообщения), медиа-флуда и массовых тегов.

### 2. Анти-скрытая реклама (Anti-Stealth Advertising / Bio & Profile Promo)
* **Анализ профилей**: Проверка никнеймов и описаний профилей (Bio) на наличие ссылок (tg://, t.me, http/https) и юзернеймов каналов.
* **Анализ призывов к действию**: Детекция фраз вроде *"посмотри мой профиль, там интересный канал"*, *"вся инфа у меня на странице"* и т.д.
* **Интеграция с ИИ**: Если эвристический фильтр бота подозревает скрытую рекламу, но не уверен на 100%, сообщение отправляется в **Модуль ИИ** для контентного анализа.
* **Наказание**: Автоматическая выдача мута (по умолчанию на 30 минут, время настраивается).

### 3. Анти-бот проверка при входе (Interactive Captcha Gate)
* **Ограничение прав**: При вступлении нового пользователя в чат бот мгновенно ограничивает его права (`restrict_chat_member`), запрещая отправку любых сообщений.
* **Интерактивная капча**: Бот отправляет в чат приветственное сообщение с inline-кнопками. Пример:
  > Привет, @username! Для защиты от спам-ботов, пожалуйста, выберите правильное животное: **Лев 🦁**
  > [ 🐯 ] [ 🦁 ] [ 🦊 ] [ 🐻 ]
* **Разблокировка**: 
  * Если пользователь нажимает правильную кнопку — бот возвращает ему права на общение и удаляет сообщение с капчей.
  * Если пользователь ошибается или не проходит капчу за 60 секунд (настраивается) — бот кикает его из чата, очищая историю заходов.

### 4. Анти-фишинг и Ссылочная безопасность (Link Guard)
* **Детекция фишинга и IP-логгеров**: Бот на лету сканирует ссылки в сообщениях и сверяет их с базами данных фишинговых сайтов, IP-логгеров (Grabify и аналоги) и вредоносных доменов.
* **Анализ накрутки**: Мониторинг резких всплесков вступлений (набегов ботов) с автоматическим баном подозрительных профилей по эвристикам.

### 5. Команды ручной модерации (Moderation Commands)
Администраторы чата (с соответствующими правами) могут применять ручные санкции через текстовые команды в чате (в ответ на сообщение нарушителя):
* `/mute` [время] [причина] — ограничить отправку сообщений (например, `/mute 30m флуд`). Если время не указано — мут выдается по умолчанию на 30 минут.
* `/unmute` — снять ограничение на отправку сообщений.
* `/kick` — исключить пользователя из чата.
* `/ban` — навсегда заблокировать пользователя в чате.
* `/unban` — разблокировать пользователя.
* `/warn` [причина] — выдать предупреждение. При достижении лимита предупреждений (по умолчанию 3) пользователь автоматически получает мут на 24 часа.
* `/unwarn` — снять одно предупреждение.

## 🤖 Part 3: AI Security Analysis & Automation (ИИ)

Для работы бота на дешевых VPS все нейросетевые проверки вынесены на **Groq API**. Все типы данных (и текст, и изображения) анализирует одна и та же продвинутая модель — **meta-llama/llama-4-scout-17b-16e-instruct** (поддерживающая и текстовый, и визуальный контекст).

Для разделения лимитов (20 RPM на ключ) используется архитектура с двумя API-ключами:

### 1. Обработка текстов (Groq Text Key)
* **API Ключ 1**: Выделен под текстовые проверки. Использует глобальный ключ из `.env` или кастомный ключ чата/канала (заданный владельцем чата через ЛС).
* **Модель**: `meta-llama/llama-4-scout-17b-16e-instruct`.
* **Реализация**:
```python
from groq import AsyncGroq

client_text = AsyncGroq(api_key="CHAT_CUSTOM_KEY_1" or "GLOBAL_KEY_1")
# Анализ скрытой рекламы, фишинга или запрещенного текста
```

### 2. Обработка изображений (Groq Vision Key)
* **API Ключ 2**: Выделен под анализ изображений. Использует глобальный ключ из `.env` или кастомный ключ чата/канала.
* **Модель**: `meta-llama/llama-4-scout-17b-16e-instruct`.
* **Реализация**:
```python
from groq import AsyncGroq

client_vision = AsyncGroq(api_key="CHAT_CUSTOM_KEY_2" or "GLOBAL_KEY_2")
# Анализ картинки на наличие NSFW/запрещенного медиа-контента
```

## ⚙️ Part 4: Configuration & Customization (Конфигурация)

Управление параметрами работы бота и распределение прав. Все настройки для админов каналов и чатов осуществляются полностью интерактивно через личные сообщения (ЛС) с ботом.

### 1. Системный конфиг (`.env` файл)
Задается один раз владельцем при развертывании бота на сервере (глобальный дефолт):
* `TELEGRAM_BOT_TOKEN` — Токен бота от BotFather.
* `GROQ_API_KEY_TEXT` — Дефолтный глобальный API ключ Groq для текстов.
* `GROQ_API_KEY_VISION` — Дефолтный глобальный API ключ Groq для картинок.
* `GROQ_MODEL` — Имя используемой модели (`meta-llama/llama-4-scout-17b-16e-instruct`).
* `OWNER_ID` — Telegram ID владельца инстанса бота (глобальный суперадмин).

### 2. Интерактивная настройка в ЛС бота (PM Settings Interface)
Пользователь пишет боту `/settings` в личные сообщения.
* **Поиск ресурсов**: Бот находит чаты и каналы, в которых он является админом, а пользователь — Создателем (Creator) или суперадмином.
* **Меню управления ресурсом**: При выборе чата/канала пользователю доступны подразделы:
  * **1. Настройки защиты (Безопасность)**:
    * Включение/выключение модулей (Капча, Ссылки, Анти-накрутка, NSFW/illegal контент).
    * Регулировка времени мута и таймаута капчи (`[ + ] / [ - ]`).
  * **2. Настройки ключей Groq (Кастомный ИИ)**:
    * Разделы: `Установить текстовый ключ` и `Установить визуальный ключ`.
    * Пользователь может отправить боту в ЛС свои личные ключи Groq для этого чата/канала. Бот сохранит их в БД в зашифрованном виде. 
    * Если кастомные ключи установлены — все запросы ИИ по данному чату/каналу идут через ключи владельца чата, не затрагивая лимиты хостера бота. Если нет — идет откат на глобальные ключи из `.env`.
  * **3. Настройки уведомлений**:
    * Установка ID канала логирования (куда бот отправляет оповещения о банах/удалениях).
  * **4. Списки слов (Blacklists)**:
    * Добавление/удаление пользовательских стоп-слов для ручного фильтра текста.

### 3. Структура Базы Данных (SQLite Entity Models)
Для хранения состояния используются следующие сущности в БД:
* **ChatSettings**: Настройки модулей для каждого чата/канала.
  * `chat_id` (Primary Key)
  * `anti_bot_flood` (Boolean)
  * `anti_admin_spam` (Boolean)
  * `anti_stealth_ad` (Boolean)
  * `captcha_gate` (Boolean)
  * `link_guard` (Boolean)
  * `mute_duration_minutes` (Integer, default 30)
  * `captcha_timeout_seconds` (Integer, default 60)
  * `custom_groq_key_text` (String, nullable) — кастомный ключ Groq для текстов
  * `custom_groq_key_vision` (String, nullable) — кастомный ключ Groq для картинок
  * `alert_channel_id` (Integer/String, nullable) — ID канала для отправки логов модерации
* **Warns**: Учет выданных предупреждений пользователям.
  * `id` (Primary Key)
  * `chat_id` (Integer)
  * `user_id` (Integer)
  * `warn_count` (Integer, default 0)
  * `last_warn_date` (DateTime)
* **ActiveMutes**: Список текущих ограничений для размута по истечении времени.
  * `chat_id` (Primary Key)
  * `user_id` (Primary Key)
  * `unmute_at` (DateTime)
* **BlacklistedWords**: Стоп-слова для конкретного чата/канала.
  * `id` (Primary Key)
  * `chat_id` (Integer)
  * `word` (String)
